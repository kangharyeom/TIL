# Summary 2022.11.17

Session이란 무엇일까?

데이터를 서버에 저장하는 것 (Cookie는 Client에 저장)
쿠키에 세션 아이디를 담아서 서버에 저장한다.

세션과 쿠키의 차이

세션 
- 저장경로: 서버  
- 장점   : 신뢰할 수 있는 유저인지 서버에서 추가로 확인 가능   
- 단점   : 하나의 서버에서만 접속 상태를 가지므로 분산에 불리 사용 인원이 많아질수록 서버 성능이 떨어짐 

쿠키
- 저장경로: 클라이언트
- 장점   : 서버에 부담을 덜어줌
- 단점   : 쿠키 그 자체는 인증이 아님
